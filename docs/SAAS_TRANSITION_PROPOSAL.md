# Wow3D Pro - SaaS 전환 및 구독형 플랫폼 개발 제안서

## 1. 개요 (Vision)
현재의 **단일 업체용 수주 시스템**을 확장하여, 전국의 수많은 3D 프린팅 출력소, 공방, 시제품 제작 업체들이 월 구독료를 내고 자신만의 **"자동 견적 및 주문 페이지"**를 개설할 수 있는 **SaaS 플랫폼**으로 전환합니다.

**슬로건:** "개발자 없이 구축하는 나만의 3D 프린팅 자동 견적 홈페이지"

---

## 2. 핵심 변경 전략 (Key Strategies)

### A. 아키텍처: 멀티 테넌트 (Multi-Tenancy) 도입
현재는 하나의 설정, 하나의 주문 DB만 존재합니다. 이를 업체(Tenant)별로 격리해야 합니다.

*   **현재:** `모든 주문` -> `관리자 1명`
*   **변경:** `업체 A의 주문` -> `업체 A 관리자`, `업체 B의 주문` -> `업체 B 관리자`
*   **기술적 요구:** 
    *   모든 DB 테이블(Users, Orders, PricingPresets)에 `store_id` 컬럼 추가.
    *   API 요청 시 어떤 스토어의 요청인지 식별(Subdomain 또는 Path 기반)하여 데이터 분리.

### B. 사용자 경험: 개인화 및 화이트 라벨링 (White Labeling)
구독하는 업체가 자신의 브랜드로 서비스를 할 수 있어야 합니다.

*   **서브도메인 제공:** `samsung-print.wow3d.com` 처럼 업체별 고유 주소 제공 (Pro 플랜 이상 커스텀 도메인 지원).
*   **브랜딩 설정:** 업체별 로고, 파비콘, 테마 컬러(Primary Color), 회사 소개, 사업자 정보 설정 기능.
*   **가격 정책 커스텀:** 업체마다 보유 장비와 재료비가 다르므로, `PricingPreset`을 업체별로 개별 설정하도록 분리.

## 3. 기술적 로드맵 (Roadmap)

### Phase 1: 아키텍처 재설계 (Foundation)
1.  **Store 테이블 생성:** 업체 정보(이름, 슬러그, 소유자) 저장.
2.  **DB 스키마 마이그레이션:** 기존 테이블에 `store_id` FK 추가 및 RLS(Row Level Security) 적용 검토.
3.  **미들웨어(Middleware) 구현:** `*.wow3d.com` 접속 시 해당 서브도메인에 맞는 Store ID를 찾아 주입하는 로직 구현.

### Phase 2: 업체 어드민(Seller Admin) 구축
1.  **회원가입/입점 프로세스:** 사용자가 "내 스토어 만들기"를 통해 서브도메인을 선점하고 관리 계정 생성.
2.  **업체별 대시보드:** 현재의 관리자 페이지를 `Seller Admin`으로 전환. (내 주문만 보이고, 내 가격만 설정 가능하게)
3.  **디자인 설정 기능:** 로고 업로드, 메인 텍스트 수정 기능 구현.

### Phase 3: 슈퍼 어드민 & 결제 (Platform Admin & Billing)
1.  **슈퍼 어드민:** 전체 업체 현황, 총 매출, 통합 사용자 관리.
2.  **구독 결제 연동:** 포트원(Portone) 또는 Stripe를 연동하여 월 정기 결제 구현.
3.  **서비스 제한 로직:** 요금제에 따른 디스크 사용량, 파일 크기 제한 등.

## 4. UI/UX 변경 예시

*   **메인 랜딩 (wow3d.com):** 서비스 소개 및 "무료로 시작하기" 버튼 (플랫폼 홍보).
*   **업체 페이지 (store.wow3d.com):** 현재 구현된 견적/주문 시스템 (개별 업체 서비스).
*   **업체 관리자 (store.wow3d.com/admin):** 해당 업체의 주문 및 견적 관리.

## 5. 기대 효과
*   **수익 다각화:** 단순 용역/판매가 아닌, SaaS 구독 모델을 통해 안정적인 **MRR(월 반복 매출)** 확보 가능.
*   **시장 선점:** 아직 국내에 소규모 출력소를 위한 저렴하고 강력한 자동 견적 SaaS가 부재함.
*   **네트워크 효과:** 많은 업체가 입점할수록 플랫폼의 인지도가 상승하고 데이터가 축적됨.
