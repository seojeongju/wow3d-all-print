## ✅ 완료된 작업: 금액 계산 로직 통일 (오차 수정)

### 📋 작업 개요
사용자가 제보한 "견적 금액과 총 결제 예정 금액의 차이" 문제를 해결했습니다.
원인은 페이지별로 **환율 적용 및 반올림 순서**가 달랐기 때문입니다.

- **기존 문제 상황**: 
  - A 페이지: `반올림(달러)` * 1300 (소수점 절삭으로 인한 오차 누적)
  - B 페이지: `반올림(달러 * 1300)` (정확함)
  - 이로 인해 약 1,300원 미만의 오차가 발생하거나, 수량이 많을 경우 오차가 커짐.

- **해결**:
  - 모든 표시/계산 로직을 **`Math.round(달러금액 * 1300)`** 방식으로 통일했습니다.
  - 이제 견적 생성 시점, 장바구니, 결제 페이지 모두 동일한 원화 금액을 표시합니다.

### 🛠️ 수정 파일
- `app/cart/page.tsx`: 개별 항목 및 총계 표시 로직 수정.
- `app/checkout/page.tsx`: 개별 항목 표시 로직 수정.

## 🚀 배포 및 테스트

### 로컬 테스트
1. 견적을 생성하여 금액 확인 (예: 13,520원).
2. 장바구니에 담아서 금액 확인 (동일하게 13,520원이어야 함).
3. 결제 페이지에서 총 결제 금액 확인 (동일해야 함).

### 배포
```bash
git add .
git commit -m "fix: 금액 계산 로직 통일 (반올림 오차 수정)"
```
