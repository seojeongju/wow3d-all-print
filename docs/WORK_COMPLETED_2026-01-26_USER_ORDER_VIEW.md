## ✅ 완료된 작업: 사용자 주문 조회 개선 및 상세 보기 기능 구현

### 📋 작업 개요
사용자 대시보드(마이페이지)에서 주문 목록이 정상적으로 표시되지 않던 문제(`NaN`, `Invalid Date`)를 해결하고, 주문 상세 정보(상품 목록, 배송지 정보 등)를 확인할 수 있는 기능을 추가했습니다.

---

## 🛠️ 해결된 문제 (Bug Fix)

### 1. 주문 목록 데이터 표시 오류 수정
- **증상**: 대시보드에서 주문 날짜가 `Invalid Date`, 금액이 `NaN`으로 표시됨.
- **원인**: API가 데이터베이스의 `snake_case` 컬럼명(`order_number`, `total_amount`)을 그대로 반환했으나, 프론트엔드는 `camelCase`(`orderNumber`, `totalAmount`)를 기대함.
- **해결**: API(`app/api/orders/route.ts`)에서 데이터를 프론트엔드 타입에 맞게 매핑하여 반환하도록 수정.

---

## ✨ 새로 추가된 기능 (New Feature)

### 1. 주문 상세 조회 모달
- **위치**: 마이페이지 > 주문 내역 탭 > 각 주문 행의 우측 '눈(Eye)' 아이콘
- **기능**:
  - 주문 상태 및 상세 날짜 확인
  - 배송지 정보 (받는 분, 연락처, 주소) 확인
  - 주문 상품 목록 상세 (파일명, 출력 방식, 파일 크기, 수량, 가격)
  - 배송 메시지 확인

**파일**: `lib/types.ts`
