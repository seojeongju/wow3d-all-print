## 🎯 **테스트 목적**

Phase 2 완성 후 전체 사용자 플로우가 정상 작동하는지 확인

---

## ✅ **테스트 체크리스트**

### 1단계: 랜딩페이지 (/)

- [ ] **URL 접속**: https://wow3d-all-print.pages.dev
- [ ] 페이지 로딩 확인
- [ ] Hero 섹션 표시 확인
- [ ] "실시간 견적 받기" 버튼 확인
- [ ] 헤더에 로고, 네비게이션, "견적 받기" 버튼 확인
- [ ] 장바구니 아이콘 (빈 상태) 확인

**예상 결과**: 
- 한국어 랜딩페이지 표시
- 다크 모드 활성화
- 모든 섹션 정상 표시

---

### 2단계: 견적 페이지 (/quote)

- [ ] "견적 받기" 버튼 클릭
- [ ] 견적 페이지 로딩 확인
- [ ] 3D 뷰어 영역 확인 (검은 배경)
- [ ] 파일 업로드 영역 확인 ("Drag & drop")
- [ ] QuotePanel 확인 (FDM, SLA, DLP 방식 선택)

**테스트 파일**: `test_cube.stl` (프로젝트 루트에 있음)

#### 2-1: 파일 업로드
- [ ] `test_cube.stl` 파일 드래그 앤 드롭 OR 클릭 업로드
- [ ] 3D 모델 표시 확인 (큐브가 3D 뷰어에 표시됨)
- [ ] 회전/줌 인터랙션 확인

**예상 결과**:
- 3D 뷰어에 큐브 표시
- 마우스로 회전 가능
- 자동 센터링

#### 2-2: 견적 계산 (FDM)
- [ ] Print Method: FDM 선택 (기본값)
- [ ] Material: PLA 선택
- [ ] Infill: 20% (기본값)
- [ ] Layer Height: 0.2mm (기본값)
- [ ] Support: 체크하지 않음

**예상 견적**:
- Volume: ~8 cm³
- Surface Area: ~24 cm²
- Dimensions: 20 × 20 × 20 mm
- Estimated Price: ~$7-8
- Estimated Time: ~1-2h

---

### 3단계: 견적 저장 (비회원)

- [ ] "견적 저장" 버튼 클릭
- [ ] Toast 알림 확인: "✅ 견적 저장 완료"

**확인 방법**:
- 개발자 도구 > Application > Local Storage > 확인
- 또는 마이페이지에서 확인 (로그인 후)

---

### 4단계: 장바구니 추가

- [ ] "장바구니" 버튼 클릭
- [ ] Toast 알림 확인: "🛒 장바구니에 추가됨"
- [ ] 헤더의 장바구니 아이콘에 배지(1) 표시 확인

---

### 5단계: 장바구니 페이지 (/cart)

- [ ] 헤더의 장바구니 아이콘 클릭
- [ ] 장바구니 페이지 로딩 확인
- [ ] 추가한 견적 표시 확인:
  - [ ] 파일명
  - [ ] 출력 방식 (FDM)
  - [ ] 재료 (PLA)
  - [ ] 부피, 예상 시간
  - [ ] 단가, 소계

#### 5-1: 수량 조절
- [ ] "-" 버튼 클릭 (수량 1이면 비활성화)
- [ ] "+" 버튼 클릭 (수량 2로 증가)
- [ ] 소계 금액 업데이트 확인

#### 5-2: 항목 삭제
- [ ] "삭제" 버튼 클릭
- [ ] Toast 알림 확인
- [ ] 항목 제거 확인
- [ ] (다시 견적 페이지에서 장바구니 추가)

---

### 6단계: 로그인 시도 (미인증 상태)

- [ ] 장바구니 페이지 우측 "주문 요약" 카드 확인
- [ ] "주문하려면 로그인이 필요합니다" 메시지 확인
- [ ] "로그인" 버튼 클릭
- [ ] `/auth` 페이지로 이동 확인

---

### 7단계: 회원가입 (/auth)

#### 7-1: 회원가입 폼 확인
- [ ] "회원가입" 탭 확인 (또는 하단 링크 클릭)
- [ ] 입력 필드 확인:
  - [ ] 이메일
  - [ ] 이름
  - [ ] 전화번호 (선택)
  - [ ] 비밀번호

#### 7-2: 회원가입 실행
테스트 계정 정보:
```
이메일: test@wow3d.com
이름: 테스트 사용자
전화번호: 010-1234-5678
비밀번호: test1234
```

- [ ] 위 정보 입력
- [ ] "회원가입" 버튼 클릭
- [ ] Toast 알림 확인: "✅ 회원가입 성공"
- [ ] 자동으로 `/cart` 페이지로 리다이렉트 확인

**예상 결과**:
- 자동 로그인 완료
- 헤더에 사용자 이름 표시 ("테스트 사용자")
- 로그아웃 버튼 표시

---

### 8단계: 장바구니에서 주문하기 (/checkout)

- [ ] 장바구니 페이지에서 "주문하기" 버튼 클릭
- [ ] `/checkout` 페이지로 이동 확인

#### 8-1: 주문 폼 확인
- [ ] 받는 사람 정보 자동 입력 확인 (회원가입 정보)
- [ ] 배송지 입력 필드 확인

#### 8-2: 배송지 입력
```
받는 사람: 테스트 사용자 (자동)
전화번호: 010-1234-5678 (자동)
배송 주소: 서울시 강남구 테헤란로 123
우편번호: 06234
요청사항: 배송 전 연락 부탁드립니다
```

- [ ] 배송지 정보 입력
- [ ] 우측 "주문 요약" 카드 확인:
  - [ ] 주문 아이템 목록
  - [ ] 총 금액
  - [ ] 예상 제작 시간

#### 8-3: 주문 생성
- [ ] 하단 "주문하기" 버튼 클릭
- [ ] 로딩 중 표시 확인
- [ ] Toast 알림 확인: "✅ 주문 완료!"

---

### 9단계: 주문 완료 페이지 (/order-complete)

- [ ] `/order-complete` 페이지로 리다이렉트 확인
- [ ] 성공 아이콘 (초록색 체크마크) 표시
- [ ] "주문이 완료되었습니다!" 메시지 확인

#### 9-1: 주문 정보 확인
- [ ] 주문 번호 표시 확인 (예: WOW3D-20260123-XXXX)
- [ ] 주문 상태: "접수 대기중" 배지 확인
- [ ] 배송지 정보 표시
- [ ] 받는 사람 정보 표시
- [ ] 총 금액 표시

#### 9-2: 다음 단계 안내
- [ ] 4단계 프로세스 표시 확인:
  1. 주문 확인
  2. 결제 안내
  3. 제작 시작
  4. 배송

---

### 10단계: 마이페이지 (/my-account)

- [ ] "주문 내역 보기" 버튼 클릭
- [ ] `/my-account` 페이지로 이동 확인

#### 10-1: 마이페이지 확인
- [ ] 프로필 정보 카드 확인:
  - [ ] 이름
  - [ ] 이메일
  - [ ] 전화번호

- [ ] 저장된 견적 카드 확인:
  - [ ] 저장한 견적 목록 표시
  - [ ] 파일명, 출력 방식, 가격, 날짜

- [ ] 주문 내역 카드 확인:
  - [ ] 방금 생성한 주문 표시
  - [ ] 주문 번호
  - [ ] 상태 (접수 대기중)
  - [ ] 날짜
  - [ ] 금액

---

### 11단계: 로그아웃 및 로그인 테스트

#### 11-1: 로그아웃
- [ ] 헤더의 로그아웃 버튼 클릭
- [ ] Toast 알림 확인: "✅ 로그아웃 완료"
- [ ] 헤더가 "로그인" 버튼으로 변경 확인

#### 11-2: 재로그인
- [ ] 헤더의 "로그인" 버튼 클릭
- [ ] `/auth` 페이지 이동
- [ ] 로그인 탭 선택
- [ ] 이메일/비밀번호 입력:
  ```
  이메일: test@wow3d.com
  비밀번호: test1234
  ```
- [ ] "로그인" 버튼 클릭
- [ ] Toast 알림 확인: "✅ 로그인 성공"
- [ ] 마이페이지로 이동 확인
- [ ] 기존 주문 내역 유지 확인

## 🔍 **추가 테스트 (엣지 케이스)**

### A. 비회원 장바구니 → 로그인 후 유지
1. [ ] 로그아웃 상태에서 견적 생성 및 장바구니 추가
2. [ ] 로그인
3. [ ] 장바구니에 아이템 유지 확인 (비회원 세션 ID 연동)

### B. 빈 장바구니에서 주문 시도
1. [ ] 장바구니 비우기
2. [ ] URL로 직접 `/checkout` 접속
3. [ ] `/cart`로 리다이렉트 확인
4. [ ] "장바구니가 비어있습니다" 메시지 확인

### C. 다양한 출력 방식 견적
1. [ ] FDM - 다른 재료 (ABS, PETG, TPU)
2. [ ] SLA - 다른 레진 (Tough, Clear, Flexible)
3. [ ] DLP
4. [ ] 각각 견적 저장 및 장바구니 추가
5. [ ] 여러 아이템 동시 주문

### D. 수량 변경 테스트
1. [ ] 장바구니에서 수량 증가 (1 → 5)
2. [ ] 총 금액 및 예상 시간 업데이트 확인
3. [ ] 주문 페이지에서도 반영 확인
